HR担当者から「Claude Codeを使って採用データを分析したい」という相談が増えている。

しかし、「とりあえずデータを渡して分析して」というアプローチでは有用な結果が出ない。この記事では、Claude CodeをHR・採用データ分析に使う前に整理しておくべき3つの前提と、実際に機能する使い方を解説する。

## 前提1：分析の「問い」を先に決める

Claude Codeに採用データを渡す前に、「何を知りたいか」を日本語で1文にまとめておく。

**機能する問いの例：**
- 「書類選考を通過した候補者と、最終面接で落ちた候補者には、経歴の違いがあるか」
- 「面接辞退が多いポジションはどれか。辞退理由に共通点はあるか」
- 「内定承諾率が低い時期はいつか。選考ステップのどこで離脱が多いか」

**機能しない問いの例：**
- 「採用データを分析して改善点を教えて」
- 「何か面白い傾向を見つけて」

問いが曖昧だと、Claude Codeが返す分析も曖昧になる。「採用プロセスのどこを改善したいか」から逆算して問いを作る。

## 前提2：個人情報を除去してからデータを渡す

採用データには候補者の個人情報が含まれる。Claude Codeに渡す前に：

1. 氏名・メールアドレス・電話番号を削除する
2. 候補者IDは連番で置き換える（「候補者001」など）
3. 住所・生年月日など個人特定につながる情報を削除する

個人情報保護法の観点から、必要最小限の項目だけで分析できるデータを渡す。「分析に必要な列だけ抽出してからClaude Codeに渡す」という習慣をつける。

## 前提3：分析結果を「答え」として使わない

Claude Codeが出した分析は「問いを立てる材料」として使う。

たとえば「書類選考通過率がエンジニアポジションで低い」という分析が出た時、それを「だからエンジニア採用は難しい」という結論にしない。

「なぜ低いのか」を採用担当者が考える。選考基準が厳しすぎるのか、応募者の経歴が求める像と合っていないのか、書類の評価方法に問題があるのか——この判断は採用担当者にしかできない。

Claude Codeはデータを整理して傾向を見せることができる。「なぜ」と「どう改善するか」は人間が担う。

---

## 実際のClaude Codeへの渡し方

```
採用データのCSVがあります。
列：応募日、ポジション名、選考ステータス（書類選考/一次面接/最終面接/内定/不合格/辞退）、辞退理由（自由記述）、内定日
個人を特定する情報は除去済みです。

知りたいこと：
・選考ステータスが「辞退」のデータで、辞退理由を分類してほしい
・辞退が多いポジション上位3つを教えてほしい

データ：
[CSVデータをここに貼り付け]
```

「データの構造」と「知りたいこと」を明確にして渡すのが基本形。
